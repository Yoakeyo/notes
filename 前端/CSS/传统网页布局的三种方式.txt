网页布局第一准则:
	多个块级元素纵向排列找标准流，多个块级元素横向排列找浮动

标准流(普通流/文档流)
	最基本的布局方式
	就是按照标签的默认方式排列
		块级元素
			独占一行，从上到下排序
			常用元素:
				div、hr、p、h1-h6、ul、ol、dl、form、table
		行内元素
			从左到右排序，碰到父元素自动换行
			常用元素:
				span、a、i、em等


浮动(float):
	用于创建浮动框，将其移动到一边
	语法:
		float:属性值;
			none	#元素不浮动(默认值)
			left	#元素向左浮动
			right	#元素向右浮动
	特性(重难点):
		1.浮动元素会脱离标准流(脱标)
			浮动的盒子不再保留原先的位置
		2.浮动的元素会在一行内显示并且元素顶部对齐
			如果多个盒子都设置了浮动，他们将会按照属性值在一行内显示
			他们是互相贴靠在一起的(不会有缝隙)，如果父级宽度装不下，多出来的会在另一行对齐
		3.浮动的元素会具有行内块元素的特性
			无论是什么元素，加了浮动之后都会具有行内块元素的特性
	通常是一个标准流的盒子内使用浮动的子盒子
	如果一个盒子浮动了，其余的兄弟元素也应该浮动
	浮动的盒子只会影响浮动盒子后面的标准流，不会影响前面的标准流
	清除浮动:
		就是清除浮动元素造成的影响
		由于父级盒子很多的情况下，不方便给高度，但是子盒子浮动又不占有位置，最后父级盒子高度为0，就会影响下面的标准流盒子
		语法:
			clear:属性值；
				left	#不允许左侧有浮动元素(清除左侧浮动影响)
				right	#不允许右侧有浮动元素(清除右侧浮动影响)
				both	#同时清除左右两侧浮动的影响
		方法:
			1.额外标签法(隔墙法),W3C推荐做法
				在浮动元素末尾添加一个空的标签。例如<div style="clear:both;" ></div>,<br />等，必须是块级元素	
			2.父级元素添加 overflow 属性
				为父级添加 overflow 属性，可以设置为hidden、auto或scroll
				无法显示溢出的部分
			3.父级元素添加 after 伪元素
				相当于额外标签法的升级版，也就是给父元素添加
					.clearfix:after {
						content:"";
						display:block;
						height:0;
						clear:both;
						visibility:hidden;
						
					}
					.clear {
						*zoom:1;	#照顾ie6、ie7
					}
			4.父级元素添加双伪元素
				也就是给父元素添加
					.clearfix:before,
					.clearfix:after {
						content:"";
						display:table;	
					}
					.clearfix:after {
						clear:both;
					}
					.clearfix {
						*zoom:1;	#照顾ie6、ie7
					}
					
定位
	当标准流和浮动都无法快速实现，就使用定位来实现
	定位 = 定位模式 + 边偏移
	定位模式指定元素的定位方式，边偏移决定了元素的最终位置
	1.定位模式
		通过position书信设置，其值可分为四个
		static			静态定位
		relative		相对定位
		absolute		绝对定位
		fixed			固定定位
	2.边偏移
		top				上
		bottom			下
		left			左
		right			右
